    Reescrever `LightField.tsx` como engine pseudo-3D. Props: `{ progress: MotionValue<number>; active: boolean }`.

    **Modelo de partícula (TRV-01/03):** cada partícula tem `z` (profundidade), `angle`+`radius` (posição polar a partir do Foco de Expansão central), `depthBucket` (índice 0..N-1 pra ordem de desenho), `size` base, `accent` (boolean, raro). Foco de Expansão = centro do canvas `(w/2, h/2)`, ESTÁVEL (TACC-02 — nunca pan/rotaciona o FoE).

    **Projeção (a assinatura da profundidade):**
    ```
    const focal = 320;                  // distância focal (px-ish), ajustável no budget
    const scale = focal / (focal + z);  // perto (z↓) = grande; longe (z↑) = pequeno
    const size = part.size * scale;      // tamanho ∝ 1/z
    const px = cx + Math.cos(angle) * radius * scale;  // expansão radial do FoE
    const py = cy + Math.sin(angle) * radius * scale;
    const alpha = f(scale);             // longe = dim (perspectiva atmosférica básica)
    ```

    **Optic flow = avanço (TRV-01):** mapear o progress pra um deslocamento de `z` do campo inteiro: `const zShift = progress * Z_TRAVEL;` e usar `const zEff = (part.z - zShift); ... wrap` — quando `zEff` passa a câmera (`zEff < zNear`), RECICLAR a partícula de volta ao fundo (`zEff += Z_RANGE`, pequena/dim). Isso dá expansão radial contínua = "estou entrando", sem custo de spawn. Em reduced-motion (plano 04) este fluxo será desligado; aqui autorar o fluxo padrão.

    **Oclusão por baldes (TRV-03 — NÃO sort por frame):** pré-particionar partículas em ~6 `depthBuckets` por z; a cada frame desenhar buckets do mais distante (z grande) pro mais próximo (z pequeno). Perto pinta por cima de longe = oclusão de graça. (Painter's algorithm com baldes, O(n), não O(n log n).)

    **Parallax em camadas (≤0.5, TACC-02):** atribuir cada partícula a 1 de 3 camadas com fator 0.3/0.4/0.5; o fator multiplica um leve deslocamento lateral proporcional ao progress (sutil). NUNCA >0.5.

Coordenadas inteiras no drawImage (`Math.round`).
