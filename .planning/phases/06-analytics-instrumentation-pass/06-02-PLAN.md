<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → vendor (GA4/Pixel/Clarity) | payloads `section_view`/`scroll_depth` saem do browser |
| usuário → form (PII) | nome/WhatsApp digitados no form podem ser capturados por gravação de sessão |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-03 | Information Disclosure | payload de section_view/scroll_depth | mitigate | Payloads carregam só `section` (literal de código) e `depth` (número) + `event_id` — zero PII. Nenhum campo de usuário é adicionado ao payload. |
| T-06-04 | Information Disclosure | gravação Clarity do form | mitigate | `data-clarity-mask="true"` no `<form>` e `<section>` (Task 3); masking client-side — conteúdo nunca chega ao servidor Clarity. |
| T-06-05 | Tampering | `section` como input | accept | `section` é literal passado por `page.tsx`, nunca input de usuário — sem superfície de injeção (ASVS V5 parcial). |
| T-06-06 | Denial of Service | scroll listener | mitigate | Listener `passive: true` + RAF throttle — coalesce para 1x/frame, não bloqueia main thread (INP protegido). |
</threat_model>
