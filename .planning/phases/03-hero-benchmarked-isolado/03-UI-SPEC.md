## Spacing Scale

Declared values (multiples of 4 — Tailwind v4 default `--spacing` step is `0.25rem` = 4px, so `p-1`..`p-16` map 1:1):

| Token | Value | Usage in Hero |
|-------|-------|---------------|
| xs (`*-1`) | 4px | Icon-to-label gap inside `<WhatsAppCta>` (already baked into `Button` `gap-2`); micro-card internal padding tight edges |
| sm (`*-2`) | 8px | Gap inside the micro-card (icon → text); border-radius offsets |
| md (`*-4`) | 16px | Default vertical rhythm between trust signal and CTA; micro-card padding (`p-4`) |
| lg (`*-6`) | 24px | Gap between sub-headline and CTA block; Container horizontal padding mobile |
| xl (`*-8`) | 32px | Gap between H1 and sub-headline; Container horizontal padding desktop |
| 2xl (`*-12`) | 48px | Gap between copy column and mockup column (desktop split); vertical breathing room around the whole Hero block on mobile |
| 3xl (`*-16`) | 64px | Hero outer vertical padding (`py-16` mobile, `py-24` desktop allowed via `lg:py-24` = 96px which is also a multiple of 4) |

**Exceptions:**
- **Mockup shadow blur radius** is a visual property, not spacing — measured separately (see Color/Effects).
- **Hero minimum height** uses `min-h-svh` or `min-h-dvh` (NOT `vh`, per HERO-05) — not a spacing token, a viewport unit. Executor calibrates so CTA sits above the fold on iPhone SE (375×667 viewport).
- **Tap targets** for `<WhatsAppCta variant="primary">` use `Button size="lg"` (`h-12` = 48px height) — meets MOBILE-03 (≥44px) preemptively even though that requirement is audited in Phase 7.
- **Mockup width** desktop = ~55–60% of viewport (D-02). On Tailwind grid: `grid-cols-1 lg:grid-cols-[1fr_1.3fr]` or similar — calibrated empirically by executor.

## Color

The Hero lives on a **dark editorial base** (D-01) with a **clear mockup "window"** (D-04). 60/30/10 split applied to the dark-dominant canvas:

| Role | Value | Usage in Hero |
|------|-------|---------------|
| Dominant (60%) | `surface-darker` `#050506` + `surface-dark` `#0a0a0b` (vertical gradient base) | Hero background canvas — full bleed dark editorial atmosphere |
| Secondary (30%) | `surface-lighter` `#ffffff` (mockup "window" surface) + `text-on-dark-primary` `#fafafa` (H1 + CTA label) + `text-on-dark-secondary` `#d4d4d8` (sub-headline) | Mockup interior + primary text on dark background + micro-card surface |
| Accent (10%) | `accent-primary` `#7c3aed` (CTA fill) + `accent-glow` `oklch(0.62 0.21 295 / 0.18)` (ambient glow behind mockup) | **Reserved-for list below** |
| Destructive | (not applicable — Hero has zero destructive actions) | — |

**Accent reserved for (explicit, never "all interactive elements"):**
1. Primary CTA button fill (`<WhatsAppCta variant="primary">` → `Button variant="default"` → `bg-accent-primary`)
2. Secondary CTA button border + text (Header `<WhatsAppCta variant="secondary">` → `border-accent-primary text-accent-primary`)
3. Ambient glow halo behind the mockup (`accent-glow` token — single pulsing layer, D-15)
4. Focus ring (`focus-visible:ring-accent-primary`) — already baked into `<Button>` (Phase 1)

**Explicitly NOT reserved for accent (Phase 1 D-03 brand-lock — mechanically blocked via Tailwind v4 token absence):**
- Section backgrounds, card fills, gradients spanning >25% of viewport, headline color, body text, decoration lines/grid, micro-card surface, sub-headline color.

**Atmosphere effects (D-06, D-15):**
| Effect | Specification |
|--------|---------------|
| Base gradient | `bg-gradient-to-b from-surface-darker via-surface-dark to-surface-darker` (vertical, neutral, almost imperceptible per D-06) |
| Grid/lines | CSS `background-image: linear-gradient(...)` at `opacity-[0.03]` to `opacity-[0.05]` — extremely subtle abstract editorial texture, NOT sci-fi dashboard (D-06) |
| Ambient glow | Single radial gradient using `accent-glow` token (`oklch(0.62 0.21 295 / 0.18)`), positioned absolutely behind the mockup, large blur radius (executor calibrates ~200–400px), `@keyframes` pulse cycle **8–12s** (D-15), scale 1.0 ↔ 1.05, opacity oscillation 0.15 ↔ 0.20, easing `--ease-premium-out` (cubic-bezier(0.16,1,0.3,1)) |
| Reduced motion | `@media (prefers-reduced-motion: reduce)` → pulse animation removed; glow remains as static layer at mid-opacity. Already handled globally by Phase 1 `globals.css` (`animation-duration: 0.01ms`), but executor confirms the static glow still visually anchors the composition |
| Mockup shadow | `shadow-2xl` or custom — large blur (~60–80px), low opacity (~0.25), offset (0, 20px), color `oklch(0 0 0 / 0.4)` over the dark base — refined sombra grande difusa (D-04) |
| Mockup border | `1px solid var(--color-border-on-dark-subtle)` `oklch(0.22 0 0)` — keeps the "window" feeling crisp without harsh contrast (D-04) |
| Mockup radius | `--radius-lg` = 12px (brand book) |
| Micro-card | `surface-lighter` background + `--radius-md` (10px) + `shadow-lg` own shadow + `border-subtle` 1px — sits OVER the mockup, slight negative offset (D-05) |

**Contrast verification (preemptive — Phase 7 audits formally):**
- `text-on-dark-primary` `#fafafa` over `surface-dark` `#0a0a0b` → ~18.5:1 (passes AAA)
- `text-on-dark-secondary` `#d4d4d8` over `surface-dark` `#0a0a0b` → ~12.6:1 (passes AAA)
- `text-on-dark-muted` `#a1a1aa` (trust signal) over `surface-dark` `#0a0a0b` → ~6.4:1 (passes AA)
- White text over `accent-primary` `#7c3aed` (CTA) → ~6.1:1 (passes AA for normal text, AAA for large)

## Interaction & Motion Contract

> Hero is **the LCP page** — strict motion discipline per HERO-02, HERO-03, MOTION-07, MOTION-08.

| Element | Initial Render | Interaction State | Reduced Motion |
|---------|----------------|-------------------|----------------|
| H1 + sub + CTA + trust signal | **Renders final state immediately. ZERO entrance animation.** (HERO-02 — LCP element protection) | CTA: `hover:bg-accent-hover hover:shadow-md active:translate-y-px` (already in `Button` default variant — premium-out easing 200ms) | No change — already static at render |
| Mockup `<Image>` | **Renders final state immediately. ZERO entrance animation. ZERO parallax. ZERO tilt.** (HERO-02/03/04 — single `priority` image of the page) | None | No change |
| Micro-card overlay | **Renders final state immediately. ZERO entrance animation.** (D-05 — even though secondary, no animation to avoid competing with protagonist) | None (static decorative chip) | No change |
| Ambient glow halo | **CSS `@keyframes` pulse 8–12s, scale 1.0↔1.05, opacity 0.15↔0.20, `--ease-premium-out`** (D-15 — only animation in Hero) | None | **Animation removed**; static glow at mid-opacity remains (D-15 + MOTION-07). Phase 1 `globals.css` already enforces `animation-duration: 0.01ms` globally under `prefers-reduced-motion`. |
| Header (logo + CTA) | Renders static, top of viewport | CTA hover/active per `<Button secondary>` baseline | No change |
| Page scroll | Lenis smooth scroll active globally (Phase 1 `SmoothScrollProvider`) | Mobile: `smoothTouch: false` (FOUND-08); desktop: `smoothWheel: true` | Lenis skips init (FOUND-08 + D-15) |

**GPU-friendly constraint (MOTION-08):** glow pulse animates ONLY `transform: scale()` and `opacity` — never `width`, `height`, `top`, `left`, `filter` (last one is debatable; executor confirms `filter: blur()` is set statically, not animated).

**Component primitives policy (Phase 2 frozen API):** Hero does NOT use `<RevealOnView>`, `<ParallaxLayer>`, `<StickyStage>`, `<TextSplit>`, or `<ScrollScene>` for any of its own elements — the glow pulse is plain CSS, not a Motion primitive. This is consistent with HERO-02/03 and Phase 2 D-15. **Audit:** `grep "motion\." src/sections/Hero/` MUST return zero matches.

## Accessibility Pre-Audit

> Phase 7 owns the formal WCAG AA audit (A11Y-01..07). Phase 3 lays clean foundations:

- H1 is a single `<h1>` on the page (SEO-08, A11Y-06 hierarchy) — `<Headline as="h1" size="hero">`

- Focus ring already declared on `<Button>` base: `focus-visible:ring-2 focus-visible:ring-accent-primary focus-visible:ring-offset-2` (A11Y-02 — visible focus)
- Mockup `<Image>` has descriptive `alt` (A11Y-07)
- Micro-card uses `role="status"` if it semantically represents a notification, OR is `aria-hidden="true"` if purely decorative — executor decides based on Lenny's chosen direction in D-05
- Decorative glow + grid are `aria-hidden="true"` and have no semantic role
- `prefers-reduced-motion` honored: glow pulse removed (D-15) — verifiable in macOS Reduce Motion / Windows Animations off (Phase 7 audit confirms formally)
