**Hard architectural rules (from 09-CONTEXT + STATE invariants):**

- ZERO `motion.div` direct import in the section. Motion comes ONLY from `@/components/motion` (`<StickyStage>`, `<ScrollScene>`). The controlled exception (README MOTION-05): inside the `<ScrollScene>` render-prop you MAY import `useTransform` + `motion.<tag>` from `motion/react` to derive sub-ranges from the emitted `MotionValue<number>`. That is the only allowed Motion import.
- Copy lives in `src/content/funnel.ts` (COPY-01 — zero hard-coded strings in JSX). Follow the `hero.ts` module shape: typed export + JSDoc header documenting locked decisions.
- Brand hex literals are forbidden in JSX/TSX (brand-lock grep test). Use brand tokens (`accent-primary`, `surface-darker`, etc.). For the dark-stage atmospheric gradients that have no token, use `rgba(124,58,237,…)` / `rgba(…)` literals exactly as the Hero does (rgba is not caught by the brand-lock grep and is the established escape hatch for cinematic dark surfaces — see `Hero/index.tsx`).
- Animate ONLY `transform` + `opacity` (MOTION-08). The prototype animates `left` on the protagonist card — **this is NOT permitted in production.** Port the horizontal travel as `transform: translateX(...)` driven by `useTransform(progress, ...)`, never `left`.

## Typography

Inter only. Two weights max for this chapter: **regular (400)** and **semibold (600)** — matches brand book (3-weight global budget; medium 500 used only on the closing statement, which is shared with the Hero's resolve-line register). Sizes use `clamp()` for fluid scaling exactly as the prototype, mapped to brand tokens.

**Size scale — exactly 4 distinct sizes** (2 body + 2 heading registers):

| # | Size | Roles |
|---|------|-------|
| 1 | `11px` | Card channel, confirmation tag |
| 2 | `14px` | Eyebrow, column head, card name, card moment (the human beat) |
| 3 | `clamp(24px, 3.4vw, 40px)` | Headline (h2) — heading register |
| 4 | `clamp(20px, 2.6vw, 30px)` | Closing statement — heading register |

The prototype's three micro-sizes (11 / 12 / 12.5px) sat in an indistinct 1.5px band. Production collapses them onto two clean steps: the 12 / 12.5px roles (eyebrow, column head, card moment) lift to **14px** (same step as the card name), and the true micro-text (channel, tag) stays at **11px**. This keeps the prototype's hierarchy readable while declaring an unambiguous 4-size scale.

## Color

This is a **DARK chapter**. The 60/30/10 split here is read against the dark stage, not the page's light baseline.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `funnel-stage` #0d0e16 + 6% purple top-radial atmosphere | The stage background the whole board sits on |
| Secondary (30%) | `funnel-column` #13141f / line #1e2030; ghost cards #0f1019 / line #181a26 @ 0.32 opacity; protagonist card #171826 / line #2a2c40 | Columns, ghost cards (the other, dimmed patients), the protagonist card in its non-climax states |
| Accent (10% → here, ~2%) | `accent-primary` #7C3AED (token) / soft-violet #a78bfa via `accent-on-dark` | Climax ONLY — see reserved list |
| Destructive | not applicable | No destructive actions in this section |

**Accent (purple #7C3AED) is reserved EXCLUSIVELY for the climax** — it is ONE of the 4 total purple moments on the entire page (09-CONTEXT D-4, milestone purple discipline). Purple appears on:

1. The "Consulta marcada" column when active (border `rgba(124,58,237,0.85)`, fill gradient `rgba(124,58,237,0.14)→0.03`, glow `0 24px 60px -20px rgba(124,58,237,0.55)`).
2. The protagonist card when it arrives at the climax (`.win` state: border `rgba(157,107,240,0.8)`, ring `rgba(124,58,237,0.4)`, violet shadow).
3. The card's confirmation tag + check seal (soft-violet text + `accent-primary` check dot).
4. The active-column dot at climax (soft-violet with `0 0 10px` violet glow).

**Purple is FORBIDDEN** on: intermediate columns 1–3 (their active state uses a NEUTRAL highlight — see below), the headline, the ghost cards, the stage background as a fill. The 6% top-radial atmosphere wash is the only ambient purple and does NOT count as a "moment" (it is texture, sub-threshold).

**Intermediate active-column treatment (NO purple):** when columns 1–3 are the active step, they brighten via NEUTRAL means only — border lifts to `rgba(124,58,237,0.45)`? **NO.** The prototype's `.column.active` uses a faint purple border at 0.45; for production we OVERRIDE this to a neutral lift to keep purple truly reserved for the climax: active border → `rgba(255,255,255,0.18)`, active head text → `text-on-dark-primary`, active dot → `text-on-dark-secondary`. The purple build is saved entirely for column 4. (This is the one deliberate tightening of the prototype to honor the locked purple-discipline; flag for Lenny: prototype showed faint purple on intermediate active columns — production removes it. If Lenny wants the faint purple back, it is a one-line change, but it weakens the climax.)

**Contrast / AA:** all text-on-dark pairs use the existing AA-validated tokens (`text-on-dark-primary` #F5F7FA, `secondary` #B6BDC9, `muted` #7F8896). The soft-violet `accent-on-dark` #a78bfa is the AA-safe purple for text on dark (ratio ~7.2, per `globals.css`) — use it for the eyebrow, tag, and closing accent span; never use raw #7C3AED for TEXT on the dark stage.

## Layout & Motion Contract

**Structure:** `<StickyStage length="560svh">` (prototype used 560vh; production MUST use `svh`) wrapping a `<ScrollScene offset={["start start", "end end"]}>`. The scene's emitted `progress: MotionValue<number>` (0→1) drives the entire journey. Inside the render-prop, derive everything via `useTransform`.

**The journey (progress 0→1):**
- The protagonist card ("Marina") travels left→right across 4 columns. Segment the progress into 3 equal travel legs (col0→1, 1→2, 2→3) exactly like the prototype's `seg = p*3` logic, but expressed as `translateX` keyframes via `useTransform`, NOT `left`. Easing per leg: easeInOut (the prototype's `t<.5 ? 2t² : 1-(-2t+2)²/2`). The README's canonical easing `cubic-bezier(0.16,1,0.3,1)` is for reveals; for the per-leg traversal use the prototype's symmetric easeInOut so each leg feels like a deliberate footstep (Lenny approved "uma pessoa caminhando").
- The active column = nearest column to the card's current position; active column brightens (neutral, per §Color), others dim to resting.
- The card's `moment` content (channel label + human beat text) swaps as the active step changes. Content from `funnel.ts` (4 beats). Swap is instantaneous content change (no per-frame React state churn — drive via discrete `useTransform` thresholds or a derived index, mirroring the Hero's `progress.on("change")` pattern; do NOT setState every frame).
- At the final leg (arrival in column 4), the climax purple ignites: column `.win.active` + card `.win` + confirmation tag fades in (`opacity 0→1`, `translateY 4px→0`). This is the felt "it happened" beat.
- Vertical position of the card: pinned just under the column header region (prototype `top: 120px`) — express as a static offset, not animated.

**Rhythm:** the prototype's ~6s easeInOut traversal feel was explicitly approved ("achei o ritmo ótimo"). The 560svh stage length is what produces that pace at typical scroll speed — keep it. On mobile, see §Responsive for the length adjustment.

**Ghost cards:** static, `opacity 0.32`, never move, never animated. They are the "other patients" — present but dimmed, proving Marina is the story and the board is the stage.

**Performance (do not regress LCP — STATE invariant):**
- Section is below the fold; no `priority` images, no LCP element here.
- Canvas-free — this is DOM + transforms, cheaper than the Hero's canvas. Still: pause work offscreen. `<StickyStage>`/`<ScrollScene>` handle scroll subscription; the section adds no autonomous RAF clock (unlike the Hero's LightField — the Funil has no continuous micro-life requirement; if a subtle idle float is desired on the protagonist card, reuse the CSS `hero-card-float-*` keyframes already in `globals.css`, which are `prefers-reduced-motion`-gated globally).
- Transform/opacity only. No layout-triggering properties.

## Responsive

80%+ traffic is mobile (PROJECT constraint). The 4-column board at phone width is the one genuine risk and the **one open decision** (09-CONTEXT: Lenny has not decided).

**RECOMMENDED mobile pattern (flag for Lenny to confirm) — "one column in focus, board as context":**

On viewports < 640px, do NOT render 4 cramped columns side-by-side (the prototype's `repeat(4,1fr)` at 170px card width overflows and the human beat text becomes unreadable). Instead:

- Show a **horizontal rail of 4 narrow column headers** (chips/labels with the step name + dot) pinned at the top — a progress indicator the user reads at a glance.
- The **protagonist card is centered and large** (full readable width, ~280–300px), and as the scroll progresses the card's content (channel + beat) crossfades through the 4 steps while the active header chip lights up and a subtle connecting line/position marker advances along the rail.
- The climax still ignites purple on the final chip + the card.
- This keeps the human beat legible (the whole point — "a pessoa caminhando", not a dashboard), preserves the left→right progression metaphor via the header rail, and avoids 4-column squash.

**Rejected mobile alternative:** "board miniaturizado" (shrink all 4 columns to fit). Rejected because the human beat text becomes illegible and it reads as a tiny dashboard — directly violating the "story not dashboard" lock. Documented here so the planner doesn't reconsider it silently.

**Desktop (≥ 640px / ≥ 1024px):** the full 4-column board exactly as the approved prototype, `max-width: 1240px`, centered, `gap-5` (20px), `px-8`.

**Mobile stage length:** reduce sticky `length` from 560svh to ~`420svh` on mobile (4 content swaps need less travel distance when the card is stationary and content crossfades) — tune at build to preserve the approved ~6s pace. Use `useDeviceTier()` to branch the length value (the only DOM/length divergence; do NOT duplicate the component).

## Accessibility

- Section `<section id="funnel" aria-labelledby="funnel-headline">`; `<h2 id="funnel-headline">` holds the headline. Eyebrow is a `<p>`/`<span>`, not a heading.
- The board, ghost cards, dots, and atmospheric layers are decorative scaffolding for a single narrative: mark the visual board container `aria-hidden` is too aggressive (the step names + beats carry meaning). Instead: expose the journey as readable text for AT — the 4 step names and Marina's 4 beats should be present in the DOM and readable in order. Confirmation seal text "Consulta confirmada · quinta, 14h" must be real text, not an icon-only state.
- No keyboard-interactive elements inside the board (it is a scroll-driven illustration). If a closing CTA is added, it follows the existing `WhatsAppCta` a11y (focus-visible ring `--color-ring`, ≥44px tap target).
- Reduced-motion path (above) is the a11y-complete fallback: the full story is present without motion.
- Color is never the sole carrier of the climax: the confirmation TAG ("Consulta confirmada · quinta, 14h" + check) carries the meaning textually, not just via purple.
